Service Blueprint

Studio Operations — Current State

A map of how work actually flows between departments today. Red markers indicate bottlenecks. Amber markers indicate manual friction. Use this as a template when documenting your own processes with staff.

How to read this blueprint

Each row is a department. Each column is a phase of the student lifecycle. Tasks are colour-coded by type:

  • Red border — bottleneck that blocks other work
  • Amber border — manual task that could be automated
  • Green border — proposed automation solution

What staff should document

For each task in your department, capture:

  • What triggers this task?
  • What information do you need to start?
  • Who do you pass the result to?
  • What slows you down or blocks you?
  • How long does it typically take?

Key Findings

Nine friction points identified across six process phases. Four are critical bottlenecks centred on the director.

Critical — Structural

Director is the single point of failure

Four of nine friction points (1, 3, 5, 8) exist because operational knowledge, decision authority, and status visibility are concentrated in one person. When the director is unavailable, operations stall.

Critical — Operational

Manual payment processing

The invoice-to-payment-to-onboarding chain (friction point 2) requires three manual handoffs across two departments. Each handoff introduces delay and error risk. This is the most impactful automation target.

Warning — Waste

Data collected but not used

Social media analytics (friction point 7) are uploaded into Lark but never analysed. Student progress data exists but isn't cross-referenced with OKRs. The studio has the data — it just doesn't flow to the right people.

Warning — Scaling Risk

No standard operating procedures

Every procedure lives in the director's memory (friction point 5). New staff can't onboard effectively. The studio cannot scale, add services, or survive the director's absence without documented processes.
